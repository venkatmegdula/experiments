Tidal Vape – Mobile Main Menu / Burger Navigation Audit

URL audited: https://www.tidalvape.co.uk/
Audit date: 2026-05-06 (UTC)
Scope: top header + main navigation + mobile/burger menu information architecture and behavior.

1) High-level website structure

  • Utility/top header: contact details, service links (Click & Collect, Careers, About Us, Visit Store), social icons, shipping message.
  • Commerce overlays: cart drawer, checkout controls, login shortcut.
  • Main navigation system: large catalog-style mega-menu with category tiles + deep multi-column links.
  • Promotional layer: deals and sale shortcuts mixed with category navigation.
  • Editorial/help links: "Help & Guidance" nested inside product categories.

2) Mobile menu behavior review (UX + Frontend)

Based on structure inspection, the menu appears to be carrying desktop mega-menu depth into mobile context. That creates cognitive load and likely interaction friction on smaller screens.

Observed bugs / gaps

SeverityObservationWhy it mattersRecommendation
High Potential duplicate links in utility/header (e.g., Careers appears repeated). Looks broken, reduces trust, and wastes scarce mobile header real estate. Deduplicate header data source and add a render guard in nav component.
High Very deep and long menu tree in mobile context (many nested lists + deal links + guides). Increases time-to-product, causes scroll fatigue, and hurts first-time discovery. Adopt 2-level mobile IA: Primary category → curated subcategory; move tertiary links to "View all" pages.
Medium Category blocks seem repeated/duplicated in menu structure. Duplication causes confusion and may indicate rendering/performance overhead. Audit nav JSON/CMS model for duplicate nodes; enforce uniqueness in build/validation step.
Medium Promotional links (Hot Sale / Multi-buy) mixed at same hierarchy as core categories. Users shopping by product type can be distracted by offer-first hierarchy. Add separate "Deals" entry point in burger menu; keep catalog taxonomy clean.
Medium Potentially inconsistent naming and typos (e.g., singular/plural forms, "Accessorie Types"). Inconsistent IA labels reduce scan speed and perceived quality. Create taxonomy glossary and enforce label linting before publish.
Low Multiple content types in one nav panel (products + brands + guides + top sellers). Not strictly broken, but dense for mobile scanning. Progressive disclosure tabs: Shop Brands Learn.

3) UX best-practice improvements for burger menu

  1. Sticky compact header: burger (left), logo (center), cart/search (right), keep row height 48–56px.
  2. Off-canvas panel: full-height drawer with clear close button + swipe-to-close.
  3. Accordion rules: only one top-level section open at a time.
  4. Search-first on mobile: put search input at top of drawer for intent-driven users.
  5. Quick links row: "Vape Kits", "E-Liquids", "Deals", "Track Order" as tappable chips.
  6. Touch targets: minimum 44px high tap areas with visible chevrons for expandable nodes.
  7. Accessibility: ensure aria-expanded, focus trap in drawer, ESC to close, and screen-reader labels.
  8. Performance: lazy-render deep children only when parent expanded.

4) Frontend implementation checklist

  • Use normalized nav data model with stable IDs and unique slugs.
  • Automated test: fail build if duplicate menu labels + same URL exist in same level.
  • Track analytics events: menu_open, menu_expand_category, menu_click_item, menu_close.
  • Add session replay markers to compare old vs redesigned mobile menu.
  • A/B test: baseline vs simplified 2-level IA and compare CTR to PDP/PLP.

5) Suggested acceptance criteria for redesigned responsive menu

  • No duplicate utility links in top header.
  • Max 8 primary menu items in burger first screen.
  • Max 2 taps to reach any high-volume product listing.
  • CLS-safe drawer open/close behavior; no layout jump.
  • Keyboard and screen-reader complete support.
